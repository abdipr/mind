# Coding

Gue suka nyoba berbagai tech stack. Di sini gue nyimpen pola-pola yang sering gue pake berulang kali.

## Aturan Ngoding Gue

1. **Keterbacaan > Pintar.** Kalo kode lu terlalu pinter, bulan depan lu sendiri yang pusing bacanya.
2. Jangan pake `any` di TypeScript kecuali terpaksa banget.
3. Tulis test buat hal yang penting, bukan buat ngejar coverage 100%.
4. Commit message yang jelas. "fix bug" itu bukan commit message.

## Stack Favorit

- **Frontend:** Next.js + React
- **Backend:** Node.js, kadang Go
- **Database:** PostgreSQL, Redis
- **Deploy:** Cloudflare Pages, Vercel

Baca juga tentang [networking](/id/tech/networking) — karena kode yang bagus nggak guna kalo servernya mati.
