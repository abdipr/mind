# Coding

I love trying different tech stacks. Here I keep patterns I use repeatedly.

## My Coding Rules

1. **Readability > Cleverness.** If your code is too clever, you'll give yourself a headache reading it next month.
2. Don't use `any` in TypeScript unless absolutely forced to.
3. Write tests for things that matter, not to chase 100% coverage.
4. Clear commit messages. "fix bug" is not a commit message.

## Favorite Stack

- **Frontend:** Next.js + React
- **Backend:** Node.js, sometimes Go
- **Database:** PostgreSQL, Redis
- **Deploy:** Cloudflare Pages, Vercel

Also read about [networking](/en/tech/networking) — great code is useless if the server is dead.
